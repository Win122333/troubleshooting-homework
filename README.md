### Анализ нагрузки потоков (Thread Dump)

На основе снятого дампа потоков был рассчитан процент занятого процессорного времени для выделения топ-3 самых нагруженных потоков.
Формула расчета: `Процент нагрузки = (cpu_ms / (elapsed * 1000)) * 100%`

| Название потока | Время жизни потока (elapsed) | Время работы потока (cpu_ms) | Процент нагрузки |
| :--- | :--- | :--- | :--- |
| C2 CompilerThread0 | 1711.74s | 14143.48ms | 0.83% |
| http-nio-8081-exec-4 | 1707.91s | 6897.79ms | 0.40% |
| http-nio-8081-exec-1 | 1707.91s | 4137.25ms | 0.24% |

**Краткий вывод:**
* **C2 CompilerThread0**: Занимает первое место. Это JIT-компилятор JVM, который переводил байт-код в машинный код и активно оптимизировал работу "тяжелого" цикла в приложении.
* **http-nio-8081-exec-4** и **exec-1**: Рабочие потоки сервера Tomcat. Они брали на себя основную работу по обработке входящих HTTP-запросов и обращений к БД H2. Низкий итоговый процент (менее 1%) объясняется тем, что нагрузка усредняется за всё время жизни потока (около 28 минут), большую часть которого сервер просто находился в режиме ожидания новых запросов (статус `WAITING`).
